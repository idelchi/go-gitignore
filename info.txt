## 1. Character Class with Backslashes (`test[\\].txt`)

The pattern `test[\\].txt` should match `test\.txt` because in a character class, `[\\]` matches a single backslash character. The issue is likely in how backslashes are processed within character classes.

## 2. Special Path `.` with Wildcard (`*`)

The test expects `.` (current directory) to be matched by `*`. However, the code has this logic:

```go
// In Git, * does not match files starting with . (dotfiles)
if strings.HasPrefix(filePath, ".") {
    return false
}
```

This is incorrect for the special case of `.` itself. In Git:
- `*` DOES match the special directory `.` (current directory)

The issue is that the code treats `.` as if it were a dotfile, but `.` as a directory reference is special and should be matched by `*`.

Here's the fix for the special path issue in `matchesFilePattern`:

```go
// Special case: * pattern should match a single path component.
// If it's rooted (/*), it should only match at root level.
if pat.pattern == wildcard {
    if pat.rooted {
        // /* should only match top-level entries
        if strings.Contains(filePath, "/") {
            return false // Not at top level
        }
        // In Git, * does not match dotfiles EXCEPT for "." itself
        if strings.HasPrefix(filePath, ".") && filePath != "." {
            return false
        }
        matched, _ := doublestar.Match(wildcard, filePath)
        return matched
    }
    // Unrooted * matches a single component at any depth
    basename := path.Base(filePath)
    if basename == "" {
        return false
    }
    // Check for dotfiles but allow "." itself
    if strings.HasPrefix(basename, ".") && basename != "." {
        return false
    }
    matched, _ := doublestar.Match(wildcard, basename)
    return matched
}
```
