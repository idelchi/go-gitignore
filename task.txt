Principle: In Git, whether a pattern matches an entry is decided on that entry (its full path if the pattern has “/”, otherwise its basename). Parent directories influence only through the parent-exclusion rule (i.e., “if any parent is ignored, descendants are ignored — unless those parents are explicitly re-included on the way down”).

Concretely:

Remove “parent basename spillover” for directories.
In your matchesFilePattern branch for patterns without “/”, keep only the basename check and delete the loop that tests ancestor basenames for directories. The match decision for a/b/c/ against b must be based on c only.

Wording/intent check.
Your comment says “Once a directory is excluded, its contents can NEVER be re-included.” That’s stronger than Git’s rule. In Git, you can re-include a deep file, but only if each parent directory on the path has also been re-included (or never excluded). Your implementation already behaves like this; just consider softening the comment so it matches behavior.
